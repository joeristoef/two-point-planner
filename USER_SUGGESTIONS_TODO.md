# User Suggestions & Feature Requests

## Core Features
- [ ] **Event Filtering Implementation** - Allow users to select specific event types/subtypes for requirement calculation (see EVENT_FILTERING_IMPLEMENTATION.md for detailed plan)
- [ ] **Collapse Staff Roster by Type** - Add ability to collapse/expand staff members grouped by staff type
- [ ] **Name Staff Members** - Allow users to assign custom names to staff members instead of generic IDs

## Data Management
- [ ] **Save/Load Filter Sets** - Allow users to save and manage filter configurations separately
- [ ] **Save/Load Staff Rosters** - Allow users to save and manage staff rosters separately

## Requirement Enhancements
- [ ] **Stat Requirements for Fantasy Staff** - Add support for stat-based requirements (not just skills and staff types)
- [ ] **Item Requirements** - Implement tracking and requirement matching for expedition items
- [ ] **Obstructions** - Research and implement obstruction mechanics from the game

## Advanced Features
- [ ] **Seasonal Expeditions** - Research and implement seasonal expedition availability/restrictions

---

## Priority Notes
- Event filtering is the main feature that will significantly increase tool usefulness
- Staff roster management (naming, collapsing) improves UX for larger teams
- Save/load functionality enables multi-scenario planning
- Item and stat requirements complete the requirement matching system
- Seasonal expeditions are a more niche feature but could enhance planning depth
